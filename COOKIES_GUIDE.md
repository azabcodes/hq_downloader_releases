This document describes how to use browser extensions to export cookies so that HQDownloader can download videos that require login credentials.

## Why Use Cookies?

Many video platforms (such as Bilibili and YouTube) require user authorization in order to:

- Watch high-resolution videos (1080P and above)
- Access premium/VIP-only content
- Download certain restricted videos

By importing your browser's cookies, HQDownloader can use your active login session to download these videos.

## Recommended Browser Extensions

### 1. Get cookies.txt LOCALLY (Recommended)

This is an open-source, secure browser extension that supports Chrome, Edge, Firefox, and other web browsers.

**Download Links:**

- Chrome/Edge: [Chrome Web Store](https://chrome.google.com/webstore/detail/get-cookiestxt-locally/cclelndahbckofkjlflfgjcfddglnfij)
- Firefox: [Firefox Add-ons](https://addons.mozilla.org/firefox/addon/get-cookies-txt-locally/)
- GitHub: [https://github.com/kairi003/Get-cookies.txt-LOCALLY](https://github.com/kairi003/Get-cookies.txt-LOCALLY)

**Features:**

- Runs completely locally; does not upload data to any server.
- Supports exporting cookies from all websites.
- Supports the Netscape format (the standard format used by yt-dlp).

## Step-by-Step Instructions

### Step 1: Install the Extension

1. Open your browser (Chrome, Edge, or Firefox).
2. Visit the download link listed above to install the extension.
3. Once installed, the extension icon will appear in your browser's toolbar.

### Step 2: Log in to the Target Website

**Important: Log in using a desktop browser!**

> ⚠️ **Key Warning**: HQDownloader video parsing requires a desktop User-Agent (Windows/Mac browser). Using a mobile User-Agent will cause extraction to fail!
>
> Therefore, make sure to log in using a **desktop browser** rather than a mobile browser.

1. Open the video website you want to download from:
   - Bilibili: https://www.bilibili.com
   - YouTube: https://www.youtube.com
   - Other sites...
2. Log in to your account.
3. Verify that you are successfully logged in (you should see your avatar or username).

### Step 3: Export Cookies

1. Click the extension icon in your browser toolbar.
2. Select **"Export All Cookies"** (or choose the specific website if you only want to export cookies for one domain).
3. Set the export format to **"Netscape"**.
4. Save the file (e.g., `cookies.txt`).

**Example Export Format:**

```
# Netscape HTTP Cookie File
# This file was generated by Get cookies.txt LOCALLY
.163.com	TRUE	/	FALSE	1811604937	_ga	GA1.2.xxx
.bilibili.com	TRUE	/	FALSE	1795412322	SESSDATA	xxx
.bilibili.com	TRUE	/	FALSE	1795412322	bili_jct	xxx
```

### Step 4: Import to HQDownloader

1. Open the HQDownloader app.
2. Go to **Settings → Cookie Management**.
3. Tap **"Import Cookie File"**.
4. Select the `cookies.txt` file you exported earlier.
5. Once imported, the app will list the websites for which cookies have been saved.

**Supported Sites:**

- Bilibili (bilibili.com)
- YouTube (youtube.com)
- Any other site supported by the parser

## Important Information

### 1. User-Agent (UA) Settings

**HQDownloader uses a desktop UA by default**, so you do not need to configure this manually.

However, if you choose to set a custom User-Agent, please note:

- Bilibili parsing **must use a desktop UA**.
- Recommended UA:
  ```
  Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36
  ```
- Do not use mobile User-Agents (iPhone, Android) as this will cause parsing failures.

### 2. Multi-Domain Cookie Support

HQDownloader supports cookies from multiple sites simultaneously:

- You can export all browser cookies at once.
- HQDownloader will automatically categorize and store them based on their domain.
- Importing cookies for Bilibili will not overwrite your YouTube cookies.

### 3. Cookie Expiration

- Most cookies expire after a certain period of time.
- If downloads start to fail, re-export and re-import the latest cookie file.
- It is recommended to check cookie validity if you run into download issues.

### 4. Privacy and Security

- Cookies contain active session data. **Never share your cookie files with others**.
- HQDownloader stores cookies locally; they are never uploaded to any remote server.
- We recommend regularly cleaning up expired or unused cookies.

## Frequently Asked Questions (FAQ)

### Q: Why can't I download HD videos even after importing cookies?

**Possible Causes:**

1. **Expired Cookies**: Re-export and import a new cookie file.
2. **Account Restrictions**: Ensure your account has the rights to watch that resolution.
3. **Mobile User-Agent**: Check if you have set a custom mobile UA in the settings.

### Q: Can I log in to multiple accounts on the same site?

No. HQDownloader uses the browser cookies as-is, meaning only one active login session can be used per site at any given time.

### Q: Can I edit the cookie file manually?

Yes, but it is not recommended. If you must edit the file, make sure:

- The Netscape format rules are preserved.
- Columns are separated using Tab characters.
- Comment lines beginning with `#` are left intact.

### Q: How do I delete imported cookies?

In HQDownloader:

1. Go to **Settings → Cookie Management**.
2. Tap the **"Clean"** button under the respective website.
3. Alternatively, tap **"Clear All Cookies"** to remove all stored cookies.

### Q: Why are YouTube downloads slow?

YouTube limits bandwidth dynamically. To optimize speed:

1. Ensure your cookies are valid and logged in.
2. Avoid queued, concurrent downloads of too many files.
3. Ensure you have a stable network connection.

## Alternative Ways to Manage Cookies

### Method 1: In-App WebView Login

HQDownloader provides a built-in login workflow:

1. Go to **Settings → Cookie Management**.
2. Tap **"Bilibili Login"** or **"YouTube Login"**.
3. Log in to your account using the in-app browser interface.
4. Cookies will be saved automatically upon a successful login.

### Method 2: Manual Cookie Text Entry

If you have the cookie string:

1. Go to **Settings → Cookie Management**.
2. Tap **"Manual Cookie Entry"**.
3. Paste the cookie text.
